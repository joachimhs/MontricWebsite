The Open Source Application Monitoring Tool!

About

Montric is an Open Source Application Monitoring Tool. As opposed to other profiling and application monitoring solutions, Montric goes out of its way in order to stay out of your existing Metric-gathering toolchain.

In a sense, Montric is Metric-source agnostic. This means that if you can measure it, you can monitor it with Montric! Montric accept metrics via a standard HTTP POST, via its JSON-based API. Once the data has been received by Montric, we won't lock you down to our services, and we wont impose a workflow on you either. You can just as easily fetch your metrics back, either as-is or aggregated and use this data in any way you'd like. This adheres to Montrics key goal, to be a truly Open Source solution from end to end.

Key Features

  • Your metrics. Your Way! Montric does not impose on either your metrics gathering toolchain, or how you retireve and use your metrics afterwards!
  • One API where you can both POST and GET your data
  • Montric keeps your data safe, while allowing you to fetch and aggregate your metrics
  • Receive alerts via Email or SMS when your applicaiton breaches your thresholds
  • Montric allows you to montior fine grained events and features within you application, right down to the response times of your important methods or functions
  • View your metrics via a consolidated view, making it easy to compare metrics against one another
  • Create chart groups where you can overlay multiple metrics onto the same chart
  • View your metrics in real time or in Historical mode
  • Set up Custom alerts for your monitored applications and services
  • Host your own Montric instance for free, or use our Montric Live Software-as-a-Service

Montric Live

Montric Live is the official hosted version of Montric, offered as Software-as-a-Service. Montric Live runs on top of dedicated hardware located in a dedicated server rack. During the beta period Montric Live is hosted on the following hardware:

  • Mars II Twin Blade pfSense Firewall (link)
  • 1x Intel Core i7 servers with 16GB RAM running Nginx for load balancing (link)
  • 2x Intel Core i7 servers with 16GB RAM each running the Montric Application
  • A 5-node Riak cluster, each node holding up to 3TB of data with a data redundancy factor of 3. Each of the Riak nodes is a seperate MicroCloud node (link)
  • Near the end of the beta perio, metrics will be backed up to Amazon S3, each hour
